*Route 53*
С помощью Route 53 можем зарегистрировать домен, а также использовать другие сервисы для работы с DNS
Используется с работы с веб доменами для разных приложений

Возможности:
1) регистрация домена
2) работа с DNS (перевод домена в ip address)
3) domain helth check

Обычно ip address явл. адресом нашего ELB

*CloudFront*
Позволяет дублировать данные в различные точки мира, чтобы была наименьшая задержка для пользователей по всеми миру
Когда пользователь запрашивает данные, CloudFront копирует их в ближайший к пользователю датацентр, кештрует и передает
Таким образом в следующий раз пользователь получит данные максимально быстро
Срок хранения данных в одной из приграничных точек составляет 24 часа

Данные сначала должны быть загружены в S3 и открыть публичный доступ для каждого из файлов которыми вы хотите поделиться
Потом выбрать существующий домен, источник  S3, настроить кэш, выбрать приграничные зоны (по умолч. выбраны все), CNAME (альтернативное имя домена (должно быть зарегистрированое в Route 53))
