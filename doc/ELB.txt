*ELB - Elastic Load Balancer*
Elastic Load Balancing – сервис, который автоматически распределяет входящий трафик приложений между несколькими инстансами Amazon EC2. ELB может выявлять упавшие инстансы и автоматически перенаправлять трафик на рабочие, пока неработающие инстансы опять не войдут в эксплуатацию, что, безусловно, повышает надежность всей системы в целом. Он дает возможность достичь отказоустойчивости в приложениях, эффективно выделяя объем ресурсов, необходимый для распределения нагрузки согласно объему входящего трафика приложений.
Не имеет free-tier

Чтобы распределить нагрузку, распределяем пользователей через ELB (пользователи используют ip-адрес ELB)


                        NACL
Subnet1 (ЕС2 --- SG) --- | ---
                         |	  |
                         |	 Route table ---Load balancer --- Intenret
                         |	  |	
Subnet2 (ЕС2 --- SG) --- | ---


Есть 3 варианта ELB:
 - Application LB (HTTP/S)
 - Network LB (TCP, TLS, UPD)
 - Classic LB (deprecated HTTP/S and TCP)

Есть 2 схемы работы:
 - внешний интернет трафик
 - внутренний трафик

ELB работает только с теми ресурсами, которые попадают в его Target group (целевую группу). Target group можно создать при создании ELB.
